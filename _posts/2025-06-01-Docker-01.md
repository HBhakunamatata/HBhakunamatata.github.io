## 2. Installation

### 2.1 Onine installation

```shell
# ubuntu24
步骤 1：更新系统并安装必要依赖
sudo apt update
sudo apt install -y apt-transport-https ca-certificates curl software-properties-common gnupg lsb-release

步骤 2：添加 Docker 官方 GPG 密钥
sudo install -m 0755 -d /etc/apt/keyrings
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg
sudo chmod a+r /etc/apt/keyrings/docker.gpg

步骤 3：添加 Docker 官方仓库
echo "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] https://download.docker.com/linux/ubuntu $(lsb_release -cs) stable" | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

步骤 4：安装 Docker
sudo apt update
sudo apt install -y docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin

步骤 5：验证 Docker 安装
docker --version
sudo docker run hello-world

可选配置：设置国内镜像加速
sudo vim /etc/docker/daemon.json

{
    "registry-mirrors": [
    "https://docker.m.daocloud.io",
    "https://hub-mirror.c.163.com"
    ]
}

允许非 root 用户运行 Docker
sudo usermod -aG docker $USER

保存后重启 Docker 服务：
sudo systemctl daemon-reload
sudo systemctl restart docker
```

### 2.2 Offline installation

```shell
# ubuntu
# 1. get deb package
wget https://download.docker.com/linux/ubuntu/dists/jammy/pool/stable/amd64/docker-ce_27.5.1-1~ubuntu.22.04~jammy_amd64.deb
wget https://download.docker.com/linux/ubuntu/dists/jammy/pool/stable/amd64/docker-ce-cli_27.5.1-1~ubuntu.22.04~jammy_amd64.deb
wget https://download.docker.com/linux/ubuntu/dists/jammy/pool/stable/amd64/containerd.io_1.7.25-1_amd64.deb

# 2. transfer into offline server and install
sudo dpkg -i containerd.io_1.7.25-1_amd64.deb
sudo dpkg -i docker-ce-cli_27.5.1-1~ubuntu.22.04~jammy_amd64.deb
sudo dpkg -i docker-ce_27.5.1-1~ubuntu.22.04~jammy_amd64.deb

# 3. config docker server
sudo systemctl start docker
sudo systemctl enable docker

# 4. check the installation
docker --version

# 5. install docker-compose
sudo dpkg -i docker-compose-plugin_2.6.0~ubuntu-jammy_amd64.deb
docker compose version
```

## 6. Application Deployment

```shell
docker run \
--name=docker-mysql-8.0 \
--restart=always \
-p 3308:3306 \
-v ~/mysql8.0/data:/var/lib/mysql \
-v ~/mysql8.0/conf:/etc/mysql \
-v ~/mysql8.0/logs:/var/log/mysql \
-v /etc/localtime:/etc/localtime \
-e MYSQL_ROOT_PASSWORD=888888 \
-d mysql:8.0

docker run \
--name=docker-mysql-5.7 \
--restart=always \
-p 3306:3306 \
-v ~/mysql5.7/data:/var/lib/mysql \
-v ~/mysql5.7/conf:/etc/mysql/conf.d \
-v ~/mysql5.7/my.cnf:/etc/mysql/my.cnf \
-v ~/mysql5.7/logs:/var/log/mysql \
-v /etc/localtime:/etc/localtime \
-e MYSQL_ROOT_PASSWORD=123456 \
-d mysql:5.7

docker run \
--name=docker-redis \
--restart=always \
-p 6379:6379 \
-v ~/myredis/conf/redis.conf:/usr/local/etc/redis/redis.conf \
-v ~/myredis/data:/data \
-v ~/myredis/logs:/var/log/redis \
-d redis

docker run \
--name=rabbitmq-docker \
--restart=always \
-p 5672:5672 \
-p 15672:15672 \
-v ~/rabbitmq/data:/var/lib/rabbitmq \
-d rabbitmq:3-management
```

## 7. dockerfile

### 7.1 docker image 原理

本质： 分层的文件系统

操作系统的文件管理系统分为bootfs和rootfs

docker镜像使用宿主机的bootfs

tomcat镜像 = rootfs + jdk + tomcat

分层复用镜像

### 7.2 镜像制作

将已经修改过的容器重新发布为一个新镜像，并导出为一个文件用于共享
被分享者使用文件恢复镜像

```shell
docker commit [container] [new_image_name]:[version]
docker save -o [file_name] [image_name]:[version]
docker load -i [file_name]
```

### 7.3 dockerfile制作方式

方法：照抄

示例：部署springboot项目

```
FROM openjdk:8-jdk-alpine
MAINTAINER macrozheng
ADD target/rabbitmq-tutorial-0.0.1-SNAPSHOT.jar /rabbitmq-tutorial-0.0.1-SNAPSHOT.jar
ENTRYPOINT ["java", "-jar", "/rabbitmq-tutorial-0.0.1-SNAPSHOT.jar"]
```
