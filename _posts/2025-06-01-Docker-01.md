# 1. Why use Docker?

The same application code behaves variously in different enviroment and depedencies.  

Application + Dependencies = Container

Docker is a container technology that solves the problem about environment migration.

# 2. What is Docker?

## 2.1 Constructure

Client --> Host --> Registry

# 3. How to use it?

## 3.1 Host

```
systemctl status | start | stop | enable docker
```

## 3.2 Images

```
docker images
docker search [image_name]
docker pull [images_name]
docker rmi [image_name]
```

## 3.3 Container

```shell
docker ps -a
docker -i -t --name nginx-1 nginx /bin/bash
docker -i -d --name nginx-2 nginx
docker exec -it nginx-2 /bin/bash
docker rm nginx-2
docker inspect nginx-1

docker start [container_name]
docker stop [container_name]
```
